# 《世界现代设计史》复习工具设计文档

## 1. 项目目标

为 2026 年 6 月 29 日的大学期末考试制作一个响应式复习网页。学生每天约有 30 分钟，考试包含选择题、填空题、名词解释、简答题和论述题。

工具部署到 GitHub Pages，在电脑和手机上使用；通过 Supabase 邮箱六位验证码登录，并同步个人复习进度。首版只服务单人考前复习，不建设教师后台、社交排行或在线 AI 出题平台。

## 2. 内容范围

题库与知识地图以用户提供的 30 页课件《世界现代设计史》为唯一考试范围来源，覆盖：

- 工业革命前设计、视觉传达设计、设计哲学、工业设计和新古典主义。
- 英国工艺美术运动、莫里斯、《红屋》及其思想局限。
- 新艺术运动及曲线型、直线型分支；威尔德、霍夫曼和德国青年风格。
- 装饰艺术运动的平面设计、美国建筑、克莱斯勒大厦及地区化发展。
- 现代主义萌起、新建筑运动和勒·柯布西耶《朗香教堂》。
- 包豪斯三个阶段、荷兰风格派、里特维德、俄国构成主义和塔特林。
- 流线型设计、人机工程学及其研究范畴。
- 国际主义设计的形成与基本特点。
- 后现代主义、英国波普、意大利激进设计和孟菲斯。

内容按照“运动—年代—理念—人物—作品—特点—影响—易混点”组织。题目答案不得加入与课件冲突的扩展知识；必要的文字纠错应保留课件原意。

## 3. 学习模型

首页只突出当天的 30 分钟任务，系统根据掌握度和复习间隔选择内容：

1. 快速回忆，约 8 分钟：抽认卡复习人物、年代、运动、代表作和关键特征。
2. 混合训练，约 15 分钟：组合五类题型，优先覆盖当天计划章节。
3. 错题回炉，约 7 分钟：重新练习答错、标记模糊或达到复习日期的知识点。

任务时长是建议值，不强制计时结束。当天完成状态、连续学习天数和总体掌握度会保存。考试倒计时固定以 2026 年 6 月 29 日为目标日期；考试日后不显示负数。

## 4. 页面架构

### 首页

- 显示考试倒计时、今日任务、完成进度和连续学习天数。
- 一个明显的“开始今日任务”入口。
- 展示当前最薄弱的三个知识点，不堆叠复杂统计图。

### 知识地图

- 一级节点按历史阶段和设计运动排列。
- 节点可展开年代、定义、核心特点、人物、代表作和易混对比。
- 支持从知识点直接进入相关抽认卡或专项练习。

### 练习

- 提供选择、填空、名词解释、简答和论述专项练习。
- 提供按题型比例组成的模拟考试。
- 每题提交后可查看答案与解析；模拟考试在交卷后统一显示结果。

### 错题

- 收录客观题答错记录，以及用户主动标记为“模糊”的主观题。
- 支持按章节和题型筛选。
- 答对并达到规定掌握度后移出当前错题队列，但保留历史记录。

### 我的

- 展示章节掌握度、学习日历、累计答题量和同步状态。
- 提供邮箱验证码登录、退出登录和手动重试同步入口。

桌面端使用左侧导航，手机端使用底部五项导航：首页、知识地图、练习、错题、我的。

## 5. 题型与判定

### 选择题

选择后自动判分。解析必须说明正确项依据，并简要指出主要干扰项错在哪里。

### 填空题

答案经过去除首尾空格、常见标点和大小写差异后比较。每题可配置多个同义答案，但不得使用宽泛的模糊匹配误判。

### 名词解释

学生先输入或口头回忆，再展开参考答案。系统显示应覆盖的得分点，例如年代、定义、核心特征、代表人物和影响。学生逐项勾选并选择“掌握、模糊、不会”。

### 简答题与论述题

学生先作答，提交后显示分点参考答案和建议结构。用户逐项勾选覆盖的得分点，系统根据勾选比例形成练习分数。该分数是复习反馈，不宣称等同教师评分。

### 掌握度

知识点掌握度综合以下信号：

- 最近客观题是否答对。
- 主观题得分点覆盖率。
- 用户选择的“掌握、模糊、不会”。
- 距离上次成功复习的时间。

未掌握内容更快再次出现；连续正确的内容逐步延长复习间隔。首版采用确定性规则，保证结果可解释和可测试。

## 6. 视觉设计

采用已批准的“编辑式复习手册”方向：

- 真白或极浅中性背景、深森林绿主色、暖棕辅助色。
- 大量留白和编辑式文字层级，适合长时间阅读。
- 少量使用包豪斯几何元素作为章节标记和进度提示，不使用高饱和三原色铺满界面。
- 避免通用 SaaS 卡片墙；内容以开放式列表、时间线、答题纸和信息带组织。
- 中文字体优先使用系统无衬线字体栈，确保 GitHub Pages 无外部字体也能稳定显示。
- 交互控件满足触屏尺寸、键盘操作、清晰焦点状态和足够对比度。
- 尊重 `prefers-reduced-motion`，动画只用于反馈和页面层级转换。

## 7. 技术架构

- 前端：React、TypeScript、Vite。
- 路由：适配 GitHub Pages 子路径的客户端路由方案。
- 测试：Vitest、Testing Library；关键响应式流程使用 Playwright 验证。
- 部署：GitHub Actions 构建并发布到 GitHub Pages。
- 登录：Supabase Auth 邮箱六位验证码。
- 云数据：Supabase Postgres。
- 本地数据：浏览器持久化存储最近状态和待同步操作。

题库作为版本控制中的结构化数据随应用发布，用户个人数据存入 Supabase。前端只使用 Supabase publishable/anon key，不包含 service role key。

若 Supabase 环境变量缺失，应用进入明确标识的本机模式：所有学习功能可用，数据仅保存在当前浏览器。配置完成并登录后，用户可开始云同步。

## 8. 数据设计

### 静态题库

- `topics`：知识点标识、章节、标题、时间范围、摘要和关联关系。
- `cards`：抽认卡正面、背面、关联知识点。
- `questions`：题型、题干、选项或得分点、答案、解析、难度和关联知识点。
- `examPresets`：模拟考试的题型数量与时间建议。

### 用户云数据

- `profiles`：用户 ID、考试日期和偏好。
- `topic_mastery`：知识点掌握度、复习次数、下次复习时间和更新时间。
- `attempts`：题目、答案摘要、结果、自评分、设备生成时间和同步时间。
- `daily_sessions`：日期、三个任务阶段的完成状态和用时。
- `mistake_items`：题目 ID、错误或模糊原因、状态和最近复习时间。

所有个人表以 `user_id` 归属用户，并启用 RLS。策略只允许 `authenticated` 角色在 `auth.uid() = user_id` 时查询和修改记录。

## 9. 同步策略

每次答题先写入本地状态并立即更新界面，再异步上传云端。无网络或请求失败时，操作进入本地待同步队列。

恢复联网、登录成功或用户点击重试时执行同步：

- 不可变的答题记录使用客户端生成的唯一 ID 去重。
- 掌握度与每日状态按 `updated_at` 采用较新的记录。
- 错题状态保留更新时间较新的操作。
- 同步失败不删除本地数据，并显示非阻断式状态提示。

用户未登录时可使用本机模式。首次登录后，系统合并本机数据与云数据，而不是直接覆盖其中一方。

## 10. 错误处理

- 邮箱格式错误、验证码错误或过期：保留邮箱并给出可执行的重试提示。
- 云端不可用：继续本地学习，显示“等待同步”。
- 题库数据异常：跳过损坏条目并记录开发错误，不能让整个今日任务白屏。
- 页面刷新或浏览器意外关闭：恢复当前未完成任务和已提交答案。
- 考试日期到达：倒计时显示“今天考试”；日期过去后显示“考试已结束”。

## 11. 验收标准

- GitHub Pages 地址可直接打开，刷新任意应用页面不会得到 404。
- 电脑与常见手机宽度均无横向溢出，核心操作可触屏完成。
- 五类题型都能完整作答、查看反馈并更新掌握度。
- 每天能生成约 30 分钟的任务，且错题和到期知识点优先出现。
- 模拟考试可组卷、交卷并显示分类结果。
- 刷新或短暂断网不会丢失已提交记录。
- Supabase 配置后，邮箱验证码可登录，同一账号跨设备看到一致的进度。
- RLS 测试证明用户不能读取或修改其他用户的数据。
- Supabase 未配置时，本机模式仍可完成全部核心学习流程。
- 内容覆盖课件的主要运动、人物、作品、概念和对比关系。

## 12. 明确不做

- 不接入在线生成式 AI 判卷或自动补充课外知识。
- 不提供用户间排行、分享、评论或公共题库编辑。
- 不制作教师后台、课程管理和多考试管理。
- 不要求安装移动应用；首版为可添加到主屏幕的响应式网页。
